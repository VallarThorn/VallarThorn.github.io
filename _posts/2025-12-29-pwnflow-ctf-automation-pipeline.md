## The Setup Tax

Every CTF pwn challenge starts pretty much the same. You first need to download an archive, extract it, figure out which file is the binary and which is the libc, run checksec, run ldd, maybe rabin2, set up a pwntools skeleton, etc. I felt this could be automated ;)

**PwnFlow** squashes all of it into a single command. The source is on [GitHub](https://github.com/VallarThorn/PwnFlow).

---

## What It Does

Drop a challenge archive in your downloads folder and run:

```bash
./pwn-pipeline --name heap_exploit --ctf picoCTF2024
```

PwnFlow runs four phases automatically:

**Phase 1 — Setup**: Extracts the archive, categorizes every file (binaries, shared libraries, source files), and creates an organized challenge directory under `~/Desktop/Ctf/picoCTF2024/heap_exploit/`.

**Phase 2 — Analysis**: Runs a full static analysis sweep on every binary and writes the results to `analysis.md`. This includes:
- `file` and basic properties (size, MD5, SHA256)
- `checksec` — NX, PIE, stack canary, RELRO, ASLR
- `ldd` for library dependencies
- `rabin2` for imports, exports, sections, and strings
- `readelf` for GOT/PLT entries, program headers, dynamic section
- `objdump` disassembly of `main()`
- Dangerous function scan: `gets`, `strcpy`, `scanf`, `system`, `exec`, `malloc`, `free`, etc.
- `pwninit` for automatic libc patching

**Phase 3 — Templates**: Generates all the boilerplate files you'd otherwise write by hand:
- `exploit.py` — a complete pwntools skeleton pre-configured with the binary, local/remote/GDB modes, and `context` settings
- `.gdbinit` — GDB init script with intel syntax, pagination off, and suggested commands on load
- `solve.sh` — quick wrapper for `./exploit.py LOCAL` or `./exploit.py HOST=... PORT=...`
- A writeup template in your Obsidian vault, pre-filled with the challenge name and structured sections for vulnerability analysis, exploitation strategy, and the flag

**Phase 4 — Cleanup**: Deletes the original archive.

## The Generated Files

The exploit template is worth showing in full — it's the file you open first every time:

```python
#!/usr/bin/env python3
from pwn import *

exe = ELF("./heap_exploit")
context.binary = exe
context.log_level = "debug"
context.terminal = ["tmux", "split-window", "-h"]

host = args.HOST or "localhost"
port = int(args.PORT or 1337)

def start_local(argv=[], *a, **kw):
    if args.GDB:
        return gdb.debug([exe.path] + argv, gdbscript=gdbscript, *a, **kw)
    else:
        return process([exe.path] + argv, *a, **kw)

def start_remote(argv=[], *a, **kw):
    io = connect(host, port)
    if args.GDB:
        gdb.attach(io, gdbscript=gdbscript)
    return io

def start(argv=[], *a, **kw):
    if args.REMOTE:
        return start_remote(argv, *a, **kw)
    else:
        return start_local(argv, *a, **kw)

gdbscript = """
continue
"""

io = start()
io.interactive()
```

Local run, remote run, and GDB attach all work out of the box with just `./exploit.py`, `./exploit.py REMOTE`, or `./exploit.py GDB`.

The Obsidian writeup template is structured for systematic analysis: security mitigations, vulnerability type and location, exploitation strategy, bypasses, step-by-step solution with code snippets, and lessons learned. It links to `analysis.md` which lands in the same vault folder.

## Debug Mode

PwnFlow has a `--debug` flag that redirects all paths to `/tmp/pwnflow/` instead of your real CTF folder and vault. Useful for testing without polluting your workspace:

```bash
./pwn-pipeline --name test_run --debug
```

Paths are all configurable via environment variables if your folder structure differs from the defaults:

| Variable | Default |
|---|---|
| `PWN_DOWNLOADS` | `~/Downloads/ctf` |
| `PWN_DEST` | `~/Desktop/Ctf` |
| `PWN_VAULT` | `~/Desktop/Vault/Ctf` |

## Installation

```bash
# Required
sudo apt install python3 file binutils coreutils findutils
pip install pwntools

# Recommended (for full analysis output)
sudo apt install checksec radare2 p7zip-full
```

The analysis script falls back when optional tools aren't available, you get whatever analysis is possible with what's installed.
